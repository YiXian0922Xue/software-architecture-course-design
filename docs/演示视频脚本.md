# LabScribe Agent 演示视频脚本（约 2 分钟）

## 0:00–0:15 场景与架构

“这是 LabScribe Agent，一个实验报告编写智能体。系统使用三层 B/S：浏览器负责交互，FastAPI 负责智能体业务，SQLite 与文件系统负责数据；外部接入 DeepSeek、Ollama 与百度 OCR。”

## 0:15–0:50 材料感知

1. 点击“新建实验项目”，输入实验题目。
2. 上传实验指导书或结果 TXT、一个 LaTeX 模板和两张实验截图。
3. 展开右侧材料栏，指出文档已索引、图片显示 OCR 字数；若 Ollama 未启动，说明系统自动使用本地嵌入降级。

旁白：“文档和 OCR 文字会被切块、嵌入并写入同一项目的 RAG 知识库。”

## 0:50–1:15 RAG 问答

输入一个只有材料中才有答案的问题，例如“本实验的采样频率和主要误差来源是什么？”展示回答和下方的文件引用、相似度与原文摘要。

旁白：“智能体不是凭空回答，而是先检索证据，再把证据交给 DeepSeek。”

## 1:15–1:45 报告生成

1. 点击“生成报告”，补充“重点分析误差来源”。
2. 生成后打开 `.tex`，快速展示章节结构与图片插入位置。
3. 展示 `_assets` 配套图片目录和源码中的 `figure`、`includegraphics`。

## 1:45–2:00 4+1 与总结

打开体系结构设计文档的 4+1 章节，快速扫过逻辑、开发、进程、物理和场景视图。

结束语：“系统体现了智能体的感知、检索决策与工具执行，并通过三层架构、RAG、Tool Use 和 Planner–Executor 模式保证可维护性与可演示性。”
